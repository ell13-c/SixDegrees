```sql
CREATE TABLE IF NOT EXISTS public.user_profiles (
    user_id        UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    display_name   TEXT NOT NULL,
    interests      TEXT[] NOT NULL DEFAULT '{}',
    location_city  TEXT NOT NULL DEFAULT '',
    location_state TEXT NOT NULL DEFAULT '',
    age            INTEGER NOT NULL,
    languages      TEXT[] NOT NULL DEFAULT '{}',
    field_of_study TEXT NOT NULL DEFAULT '',
    industry       TEXT NOT NULL DEFAULT '',
    education_level TEXT NOT NULL DEFAULT '',
    timezone       TEXT NOT NULL DEFAULT 'UTC',
    updated_at     TIMESTAMPTZ NOT NULL DEFAULT now()
);
```

3. interactions table (with composite PK and canonical pair CHECK):
```sql
CREATE TABLE IF NOT EXISTS public.interactions (
    user_id_a      UUID NOT NULL,
    user_id_b      UUID NOT NULL,
    likes_count    INTEGER NOT NULL DEFAULT 0,
    comments_count INTEGER NOT NULL DEFAULT 0,
    dm_count       INTEGER NOT NULL DEFAULT 0,
    last_updated   TIMESTAMPTZ NOT NULL DEFAULT now(),
    PRIMARY KEY (user_id_a, user_id_b),
    CONSTRAINT canonical_pair_order CHECK (user_id_a < user_id_b)
);
```

4. map_coordinates table (tier is smallint with 1/2/3 CHECK, is_current defaults true):
```sql
CREATE TABLE IF NOT EXISTS public.map_coordinates (
    id             UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    center_user_id UUID NOT NULL,
    other_user_id  UUID NOT NULL,
    x              DOUBLE PRECISION NOT NULL,
    y              DOUBLE PRECISION NOT NULL,
    tier           SMALLINT NOT NULL CHECK (tier IN (1, 2, 3)),
    computed_at    TIMESTAMPTZ NOT NULL DEFAULT now(),
    is_current     BOOLEAN NOT NULL DEFAULT TRUE
);
```

5. Composite index on map_coordinates (DB-04 — critical for Phase 4 API reads):
```sql
CREATE INDEX IF NOT EXISTS idx_map_coordinates_center_is_current
    ON public.map_coordinates (center_user_id, is_current);
```

6. Enable RLS on all three tables (service role key bypasses RLS; this prevents anon key access):
```sql
ALTER TABLE public.user_profiles ENABLE ROW LEVEL SECURITY;
ALTER TABLE public.interactions ENABLE ROW LEVEL SECURITY;
ALTER TABLE public.map_coordinates ENABLE ROW LEVEL SECURITY;
```

7. FK constraints on interactions (seed script inserts profiles first, so this order is safe):
```sql
ALTER TABLE public.interactions
    ADD CONSTRAINT fk_interactions_user_a
    FOREIGN KEY (user_id_a) REFERENCES public.user_profiles(user_id);

ALTER TABLE public.interactions
    ADD CONSTRAINT fk_interactions_user_b
    FOREIGN KEY (user_id_b) REFERENCES public.user_profiles(user_id);
```

Do NOT add FK from user_profiles.user_id to auth.users — seed users won't exist in auth.users (RESEARCH.md open question 1). The ADD CONSTRAINT statements do not need IF NOT EXISTS equivalents because the whole script is safe to re-run only if tables don't already exist — if they do, the CREATE TABLE IF NOT EXISTS no-ops, but the ADD CONSTRAINT would error. Wrap the FK block in a DO block to make it idempotent:

```sql
DO $$
BEGIN
    IF NOT EXISTS (
        SELECT 1 FROM information_schema.table_constraints
        WHERE constraint_name = 'fk_interactions_user_a'
    ) THEN
        ALTER TABLE public.interactions
            ADD CONSTRAINT fk_interactions_user_a
            FOREIGN KEY (user_id_a) REFERENCES public.user_profiles(user_id);
    END IF;
    IF NOT EXISTS (
        SELECT 1 FROM information_schema.table_constraints
        WHERE constraint_name = 'fk_interactions_user_b'
    ) THEN
        ALTER TABLE public.interactions
            ADD CONSTRAINT fk_interactions_user_b
            FOREIGN KEY (user_id_b) REFERENCES public.user_profiles(user_id);
    END IF;
END $$;
```
