Create `backend/routes/map.py` with two endpoints on an APIRouter with prefix="/map" and tag="map".

**GET /map/{user_id}:**
1. Query `map_coordinates` table for `center_user_id == user_id` AND `is_current == True`
2. If no rows found, raise `HTTPException(status_code=404, detail="Map not yet computed for this user")`
3. Collect all `other_user_id` values from the rows
4. Fetch `user_id, display_name` from `user_profiles` using `.in_("user_id", other_ids)` — single query, not N queries in a loop
5. Build a `name_map = {p["user_id"]: p["display_name"] for p in profiles}`
6. Build response:
```python
{
    "user_id": user_id,
    "computed_at": rows[0]["computed_at"],
    "coordinates": [
        {
            "user_id": r["other_user_id"],
            "x": r["x"],
            "y": r["y"],
            "tier": r["tier"],
            "display_name": name_map.get(r["other_user_id"], ""),
        }
        for r in rows
    ]
}
```

**POST /map/trigger/{user_id}:**
- No JWT required (per locked decision in CONTEXT.md — used for testing/demo)
- Call `run_pipeline_for_user(user_id)` — import from `services.map_pipeline`
- If `ValueError` is raised (N < 10 or user not found), return HTTP 422 with the error message as detail
- After successful run, fetch and return the updated coordinates using the same query + display_name lookup logic as GET (do not duplicate code — extract the lookup into a helper or inline it clearly)

**Critical notes:**
- `map_coordinates` columns: `id, center_user_id, other_user_id, x, y, tier, computed_at, is_current` — NO display_name column
- The requesting user appears in `map_coordinates` as a row where `other_user_id == center_user_id` at (0.0, 0.0) — this is guaranteed by coord_writer from Phase 3, no special handling needed
- `import run_pipeline_for_user` at the top of the file (not inside the function) — no circular import risk since map.py doesn't import from deps.py

Full file structure:
```python
from fastapi import APIRouter, HTTPException
from config.supabase import get_supabase_client
from services.map_pipeline import run_pipeline_for_user

router = APIRouter(prefix="/map", tags=["map"])


def _fetch_map_response(user_id: str) -> dict:
    """Shared helper: fetch current map_coordinates + display_names for a user."""
    sb = get_supabase_client()
    rows = (
        sb.table("map_coordinates")
        .select("other_user_id, x, y, tier, computed_at")
        .eq("center_user_id", user_id)
        .eq("is_current", True)
        .execute()
    ).data

    if not rows:
        raise HTTPException(status_code=404, detail="Map not yet computed for this user")

    other_ids = [r["other_user_id"] for r in rows]
    profiles = (
        sb.table("user_profiles")
        .select("user_id, display_name")
        .in_("user_id", other_ids)
        .execute()
    ).data
    name_map = {p["user_id"]: p["display_name"] for p in profiles}

    return {
        "user_id": user_id,
        "computed_at": rows[0]["computed_at"],
        "coordinates": [
            {
                "user_id": r["other_user_id"],
                "x": r["x"],
                "y": r["y"],
                "tier": r["tier"],
                "display_name": name_map.get(r["other_user_id"], ""),
            }
            for r in rows
        ],
    }


@router.get("/{user_id}")
async def get_map(user_id: str):
    return _fetch_map_response(user_id)


@router.post("/trigger/{user_id}")
async def trigger_map(user_id: str):
    try:
        run_pipeline_for_user(user_id)
    except ValueError as exc:
        raise HTTPException(status_code=422, detail=str(exc))
    return _fetch_map_response(user_id)
```
