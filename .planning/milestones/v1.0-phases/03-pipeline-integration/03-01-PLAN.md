  def fetch_all() -> tuple[list[UserProfile], dict[tuple[str, str], dict[str, int]]]:

2. Read user_profiles:
   response = sb.table("user_profiles").select("*").execute()
   Build list[UserProfile] from response.data (attribute access, NOT dict access — supabase-py returns an object).

   CRITICAL field mapping (DB column → UserProfile field):
   - row["user_id"]          → id=
   - row["location_city"]    → city=   (NOT "city" — will KeyError)
   - row["location_state"]   → state=  (NOT "state" — will KeyError)
   - row["interests"] or []  → interests= (guard None with `or []`)

   - row["field_of_study"]   → field_of_study=

   - row["age"]              → age=

   response = sb.table("interactions").select("*").execute()
   Build dict[tuple[str,str], dict[str,int]] from response.data.

   pair_key = (row["user_id_a"], row["user_id_b"])  # already canonical in DB (a < b)

   Value dict keys MUST match INTERACTION_WEIGHTS in config/algorithm.py:
   {"likes": row["likes_count"], "comments": row["comments_count"], "dms": row["dm_count"]}

4. Return (users, raw_interaction_counts)

Create backend/services/map_pipeline/coord_writer.py with a single public function write_coordinates().

Imports:
- from datetime import datetime, timezone
- from config.supabase import get_supabase_client
- typing: list

Function signature:
  def write_coordinates(center_user_id: str, translated_results: list[dict]) -> None:

Implementation (write order is MANDATORY — do not reverse):

Step 1 — Mark old rows as not current (STORE-01, STORE-02):
  sb.table("map_coordinates") \
    .update({"is_current": False}) \
    .eq("center_user_id", center_user_id) \
    .eq("is_current", True) \
    .execute()

  Never call .delete() — old rows are retained for future animation delta (STORE-02).

Step 2 — Build and insert new rows (STORE-03):
  now = datetime.now(timezone.utc).isoformat()
  rows = [
      {
          "center_user_id": center_user_id,
          "other_user_id":  entry["user_id"],
          "x":              entry["x"],
          "y":              entry["y"],
          "tier":           entry["tier"],
          "computed_at":    now,
          "is_current":     True,
      }
      for entry in translated_results
  ]
  sb.table("map_coordinates").insert(rows).execute()

  translated_results contains ALL users including the center user at (0.0, 0.0) — do NOT filter any entries out. The center user row will have other_user_id == center_user_id with x=0.0, y=0.0, tier=1.

Add a module docstring explaining: two-step write pattern, why DELETE is forbidden (STORE-02 animation delta), that center user at (0,0) is included in every write (STORE-03).
  </action>
